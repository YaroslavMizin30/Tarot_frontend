# P1: надёжная сессия и атомарная регистрация

## Что изменено

- При повторном открытии приложения проверенная Supabase-сессия используется повторно, если её `telegram_id` совпадает с текущим пользователем Telegram.
- Повреждённая, истёкшая или принадлежащая другому Telegram-пользователю локальная сессия очищается, после чего выполняется обычная проверка `initData`.
- Создание `users`, `activity` и `analytics` перенесено в одну SQL-функцию `complete_user_onboarding`.
- Telegram ID берётся только из проверенной Supabase-сессии. Клиент не может выбрать ID создаваемого профиля.
- Сервер проверяет имя и дату рождения, сам вычисляет знак зодиака и сериализует параллельные повторы регистрации.
- Миграция восстанавливает отсутствующие `activity` и `analytics` у профилей, созданных старым раздельным флоу.
- Если HTTP-ответ потерян после коммита, клиент перечитывает профиль. Пользователь не увидит ложную ошибку и может безопасно повторить запрос.

## Порядок деплоя

1. В backend применить миграцию `202607200001_atomic_user_onboarding.sql`.
2. Убедиться, что функция `complete_user_onboarding` доступна роли `authenticated`.
3. Только после миграции развернуть frontend этой ветки.

Edge Function для P1 не добавляется и отдельно не деплоится.

## Проверка

1. Существующий пользователь повторно открывает Mini App и попадает в приложение без регистрации.
2. Новый пользователь проходит два шага регистрации и после одного нажатия попадает на главную.
3. Повторный вызов регистрации с теми же данными не создаёт дубли в `activity` или `analytics`.
4. После перезапуска Mini App пользователь остаётся авторизован; в логах `telegram-auth` нет обязательного нового вызова при каждой загрузке.
5. При открытии Mini App другим Telegram-аккаунтом сохранённая сессия первого аккаунта не используется.

## Откат

Сначала откатить frontend на предыдущую production-версию. Миграцию можно оставить: она обратно совместима со старым клиентом и не меняет структуру таблиц.

Если функцию всё же нужно удалить после отката frontend:

```sql
revoke all on function public.complete_user_onboarding(text, text, text, text)
  from public, anon, authenticated;
drop function if exists public.complete_user_onboarding(text, text, text, text);
```

Восстановленные строки `activity` и `analytics` удалять не нужно.
